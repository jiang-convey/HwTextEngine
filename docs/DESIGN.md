# 设计文档 · HwTextEngine

本文说明各算法的思路、复杂度与在本项目中的落点，以及整体架构的取舍。代码均为
从零实现，标准库之外零依赖（PDF 解码除外，且被隔离在 `io/`）。

## 1. 总体架构

```
                   ┌─────────────────────────────────────────┐
   PDF/TXT ──io──▶ │  纯算法内核（零依赖）                      │
   公告/教材/答案   │  matching/  parsing/  compress/           │
                   └───────────────▲───────────────────────────┘
                                   │ 纯函数：吃 str，吐数据结构
                   ┌───────────────┴───────────┐
                   │  apps/  把算法串成功能      │   locate / mine / fix / analyze
                   └───────────────▲───────────┘
                                   │
                            cli.py（界面层：argparse + 打印 + --json）
```

三条分层原则：

1. **逻辑 / 界面分离**：`apps/*` 只返回数据结构，不打印；所有 I/O 与展示在 `cli.py`。
2. **算法 / 库分离**：第三方库只允许出现在 `io/`，且只做"PDF→字符串"的解码；拿到
   字符串后，检索 / 匹配 / 解析 / 压缩全部走自实现算法。这条边界让"底层实现"可被审查。
3. **可测试**：核心算法是纯函数，`tests/` 用标准库 `unittest`，47 个用例，零依赖可跑。

## 2. 字符串匹配

### 2.1 KMP（`matching/kmp.py`）— 字符串匹配
- **失配函数** `fail[i]`：`pattern[:i+1]` 的最长「真前缀 == 真后缀」长度。构造时用已算出的
  `fail` 自我匹配，均摊 O(m)。
- **匹配**：主串指针只前进不回退；失配时 `k ← fail[k-1]`。命中后同样 `k ← fail[k-1]`，
  因此能报告**重叠**出现（`aa` 在 `aaaa` → `[0,1,2]`）。
- **复杂度**：O(n+m) 时间，O(m) 空间。
- **落点**：`locate` 中搜索题号标记「习题3.2」；`mine` 中抽取「习题/第…题/截止」线索。

### 2.2 Aho–Corasick（`matching/aho_corasick.py`）— 多模式 / 自动机
- **字典树**：所有关键词插入 Trie，终止结点记 `out`。
- **失配指针**：对 KMP 思想的多模式推广。用 **BFS（层序）** 构建：结点 `u`（经字符 `c`
  到达）的 fail，是沿父结点 fail 链找到的第一个有 `c` 出边的祖先的对应孩子；再把
  `fail` 结点的 `out` 并入自身（后缀闭包），从而一次扫描能识别所有作为后缀出现的模式。
- **扫描**：主串只走一遍，失配沿 fail 链回退。
- **复杂度**：构建 O(Σ|pᵢ|)，扫描 O(n + z)（z 为命中数）。
- **落点**：`mine` 一次扫描公告即同时命中「作业/习题/截止/deadline/第/页/homework…」整批
  关键词——这是选用 AC 而非对每个词各跑一次 KMP 的原因。

### 2.3 编辑距离（`matching/edit_distance.py`）— 动态规划
- **状态**：`dp[i][j]` = `a[:i]` 变成 `b[:j]` 的最少增/删/改次数。
- **转移**：`dp[i][j] = min(dp[i-1][j]+1, dp[i][j-1]+1, dp[i-1][j-1] + [a≠b])`。
- **实现**：默认滚动数组，空间 O(min(m,n))；另给完整表版本便于可视化/回溯。
- **复杂度**：O(m·n)。
- **落点**：`locate` 精确题号找不到时，对扫描到的真实题号集合按编辑距离排序，给候选。

## 3. 解析与状态机

### 3.1 有限状态机（`parsing/tokenizer.py`）
三态 `NORMAL / INLINE / FENCED`，线性扫描得到「代码保护掩码」。公式修复与括号校验都
**先问掩码、再处理**，从而绝不动用户贴的示例代码。这是把"跳过代码块"这一横切关注点
收敛到一处的设计。O(n)。

### 3.2 栈配对（`parsing/bracket_stack.py`）— 栈
- 严格**从左到右（自然阅读顺序）**扫描：左定界符 `( [ { \(  \[` 入栈，右定界符与栈顶
  配对并出栈，结束时栈空即平衡。报告三类错误：多余右括号 / 与栈顶不匹配 / 未闭合左括号。
- `$ / $$` 是"开闭同形"的切换符，单独用奇偶状态机校验，不混进括号栈。
- **为何从左到右**：括号匹配的自然语义方向就是阅读方向，先出现的左括号最后闭合，恰是
  栈的 LIFO 语义；刻意反向扫描只会把逻辑复杂化。O(n)，栈深 O(嵌套层数)。
- **落点**：`fix` 在渲染前校验答案定界符，提前报错而非让 MathJax 静默崩坏。

### 3.3 分隔符规整（`parsing/delim_fix.py`）
基于掩码把 `\[ \] → $$`、`\( \) → $`（代码区不动），返回替换次数。O(n)。

### 3.4 递归下降 → AST（`parsing/md_parser.py`）— 树 / 递归
- 块级文法：`document := (heading | fence | math | quote | list | paragraph)*`。
- 每种块一个解析函数；**嵌套列表**按缩进**递归**调用 `_list(indent)` 构建子树——递归
  下降与树结构的直接体现。
- 行首标记（标题 `#`、列表 `- / 1.`）用**手写谓词**识别，未用正则，以保持"算法零依赖"。
- **边界**：面向作业文档的实用子集，非 CommonMark 全量；行内强调/链接不展开。O(n)。
- **落点**：`analyze` 输出结构大纲，并为"只对非代码块修分隔符"提供结构依据。

## 4. 压缩

### 4.1 二叉最小堆（`compress/heap.py`）— 堆 / 优先队列
数组表示完全二叉树（孩子 `2i+1/2i+2`，父 `(i-1)//2`），`push` 上浮、`pop` 下沉，
支持自定义 `key`。手写而非用 `heapq`，以体现底层实现。push/pop O(log n)，heapify O(n)。

### 4.2 Huffman（`compress/huffman.py`）— 贪心 / 二叉树 / 前缀码
统计频率 → 每字符建叶入堆 → 反复取两个最小频率结点合并（贪心）→ 得最优前缀码。
以 `(频率, 入堆序)` 为堆序键，保证并列时结果**确定可复现**。单字符特判（码长保证 ≥1）。
建树 O(k log k)，编/解码 O(n)。**落点**：`analyze` 报告题面文本压缩比与平均码长。

## 5. 测试

`tests/` 共 47 个 `unittest` 用例，覆盖：
- KMP 失配表 / 重叠匹配 / 与 `str.find` 交叉验证 / 空模式；
- AC 经典 `ushers` 例 / 命中位置自洽 / 去重 / 借失配指针识别后缀模式；
- 编辑距离已知值（kitten→sitting=3）/ 空串 / 对称性 / 候选排序；
- 栈：平衡 / 不匹配 / 未闭合 / 多余右括号 / `$$` 奇偶 / 代码跳过；
- 分隔符：基本改写 / 围栏与行内代码跳过 / 混合；
- 递归下降：块类型序列 / 标题层级 / **嵌套列表递归** / 代码与公式载荷 / 大纲；
- 堆：有序弹出 / 自定义 key / heapify / 空堆异常；
- Huffman：往返一致 / 前缀码无歧义 / 单字符 / 压缩比 / 空输入；
- 应用层：定位命中与截断、候选回退、公告判定与题号抽取、定界符修复、文档分析。

运行：`python -m unittest discover -s tests -p "test_*.py" -v`（零第三方依赖）。

## 6. 已知边界

- Markdown 解析为实用子集，不追求 CommonMark 完整性。
- 公告"挖作业"是基于关键词 + 题号/截止线索的启发式：判定规则为「有题号，或有强信号词且
  同时含截止线索」，以压低"本通知不涉及作业"这类误报，但仍可能漏判极不规范的通知。
- `$ / $$` 校验只做奇偶（成对性），不解析其内部 LaTeX 合法性。
- PDF 文本质量取决于解码库与原文档排版（扫描件无文本层时不可用）。
